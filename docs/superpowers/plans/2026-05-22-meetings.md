# Meetings (Phase 2) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Ship WAAO Meetings — meeting = calendar event + organizer + attendees (individuals & departments) + RSVP + structured agenda; surfaces on every attendee's calendar.

**Architecture:** New `Meeting` / `MeetingAttendee` / `MeetingAgendaItem` entities, 1:1 with `CalendarEvent`. One migration `AddMeetings`. `MeetingService` orchestrates event+meeting creation and attendee expansion. `CalendarEventService.GetOccurrencesAsync` extended to include meetings the caller attends. Frontend: meeting dialog + detail panel, calendar integration.

**Tech Stack:** .NET 9, EF Core 9, FluentValidation, React 19 + TS + Vite, TanStack Query.

**Spec:** `docs/superpowers/specs/2026-05-22-meetings-design.md` — read before starting.

**Golden examples:** Backend — `src/Waao.Services/Services/CalendarEventService.cs`, `CalendarService.cs`, `src/Waao.API/Controllers/CalendarEventsController.cs`, recent migration `20260522151729_AddCalendar.cs`. Frontend — `src/pages/calendar/event-dialog.tsx`, `recurrence-editor.tsx`, `edit-scope-prompt.tsx`, `src/components/ui/portal-menu.tsx`.

**Conventions (every task):** Backend TABS + file-scoped namespaces + primary-ctor DI (PascalCase); DTOs `record`; `Guid.CreateVersion7()`; `DateTime.UtcNow`; soft delete; enums `HasConversion<string>()`; FluentValidation. Frontend no `any`, `t()` × 3 locales, TanStack Query, WAAO UI lib, portal modals. Migrations via Bash heredoc (Write/Edit hook-blocked). Commit `git -c user.email="higor@waao.com.br"`, conventional prefix, no Claude/AI/Co-Authored-By. Push to `main`.
