# Full-Site i18n (pt-BR source) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Wire every user-visible hardcoded string in `WaaoFrontend` through `t()` against `src/locales/{pt-BR,en,es}/common.json`, with **pt-BR as the canonical source language**.

**Architecture:** i18n infra is already in place (`i18next` + `react-i18next`, `LanguageDetector`, 3 locales, single `common` namespace, `fallbackLng: 'pt-BR'`, sidebar switcher). Features B/C/D added ~70 keys already. This feature **extracts the remaining hardcoded strings** from the ~28 untranslated `.tsx` files and the partially-translated pages where B/C/D deliberately deferred. Decomposed by page-module so each task is independently reviewable/mergeable.

**Tech Stack:** React 19 + TS + Vite, `react-i18next`, `i18next`, the app's own UI lib at `@/components/ui/*`. NO `any`. pt-BR genuine source; en + es real translations (never copies of pt-BR — natural English / natural Spanish).

**Spec (design decisions captured at brainstorm):**
- pt-BR is the canonical source language; fallback is pt-BR.
- Out of scope: decorative branding (`"WAAO Journey"`, the left-panel logo/copy on `login-page`/`register-page` — those are intentional un-translated design elements, confirmed in Feature C's code review).
- In scope: functional user-visible text — page titles/headings, labels, button labels, placeholder hints, validation messages, toasts/banners, empty-state copy, link text, aria-labels for icon-only buttons.
- Key namespace structure: existing top-level objects (`app`, `nav`, `auth`, `common`, `collaborators`, `badges`, `leaderboard`, `admin`, `onboarding`, `verify`) — extend these and add new module-level top-level keys (`dashboard`, `kanban`, `docs`, `settings`, `comingSoon`, `sidebar`) as needed.

**Working repo:** `/Users/higorflopes/RiderProjects/Repositories/Waao/WaaoFrontend` (separate git repo from backend), on `main`. All standards apply: NO `any`, no `@medtrack/ui`, app's own UI components, commits conventional + NO AI/Claude/Anthropic/Co-Authored-By, `git add` ONLY task files (never `-A`/`.`; `.claude-flow/`, `dist/`, `*.tsbuildinfo` stay out). Author `git -c user.email="higor@waao.com.br"`. Do NOT push between tasks (the controller pushes after each task's reviews pass, OR a single push at A9 — controller decides per the established session pattern). Pushing the frontend triggers Cloudflare Worker CI auto-deploy. **There is no backend change in this feature — no `fly deploy` needed.**

**Per-task workflow (universal — applies to A2–A8):**
1. **Inventory:** read the target files; list every hardcoded user-visible string. Use `grep -nE '>[A-Za-zÀ-ú][^<>{}]{2,}<' <file>` + manual reading for string literals in JSX `placeholder`, `title`, `aria-label`, `toast()`, `setError({message})`, conditional empty-states, etc.
2. **Decide pt-BR canonical text** per string: if the source already reads as pt-BR (e.g. `"Salvar"`), keep it. If it's English, translate it to natural pt-BR. The pt-BR locale entry stores this canonical value.
3. **Pick keys** under the right top-level namespace (extend `dashboard`, `kanban`, etc.). Group logically: `<module>.<section>.<purpose>` (e.g. `dashboard.topPerformers.title`). Keep ordering consistent across all 3 locales.
4. **Update all 3 locale files** (`src/locales/{pt-BR,en,es}/common.json`) — pt-BR with the canonical Portuguese, en with real English (not Portuguese), es with real Spanish (not Portuguese). Validate JSON for each file with `node -e "JSON.parse(require('fs').readFileSync('src/locales/<l>/common.json','utf8'))"` (must succeed for pt-BR, en, es).
5. **Replace** each hardcoded string in the tsx with `t('<key>')`. Add `useTranslation()` if the file doesn't already import it. Preserve any interpolation by using `t('key', { name, count })` with `{{name}}` placeholders in the locale values.
6. **Build** `npm run build` — 0 TS errors, no `any`, no unused-locals.
7. **Smoke** by visual diff in any rendered page is out of scope here — the build pass + the JSON validity check is the automated gate. (No frontend unit tests exist; do NOT introduce a new test framework as part of i18n extraction — that's separate scope.)
8. **Commit** scoped to the page files + the 3 locale files.

**Out-of-scope for every task** (do NOT touch): the decorative left-panel branding on the auth pages (`"WAAO Journey"`, intro tagline); the `useCelebrations` hook internals; any `@/components/ui/*` shadcn-style primitives (Button/Card/Input/Label etc. — they expose no user-visible strings); the celebration components (`xp-chip`, `level-up-overlay`, `badge-unlock-toast`) since they were already deferred by B and contain auto-content; anything outside `WaaoFrontend/src`. Do NOT extract strings that are already i18n'd (do NOT re-key existing `t()` calls).
