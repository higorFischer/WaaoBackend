# Calendar (Phase 1) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the WAAO Calendar — personal/department/company calendars, events with full iCalendar recurrence + per-occurrence overrides, month/week/agenda views — backend + frontend + i18n + nav.

**Architecture:** New `Calendar` backend module (entities → EF config → migration → DTOs → validators → recurrence expander → services → controllers). One migration `AddCalendar`. Recurrence handled by the `Ical.Net` NuGet package wrapped in `IRecurrenceExpander`. Frontend `/calendar` route with Month/Week/Agenda views, portal-based event dialog, RRULE recurrence editor.

**Tech Stack:** .NET 9, EF Core 9 (Npgsql), Ical.Net, FluentValidation, React 19 + TS + Vite, TanStack Query, i18next.

**Spec:** `docs/superpowers/specs/2026-05-22-calendar-design.md` — read it before starting.

**Golden examples:** Backend — `AdminService.cs`, `CoursesController.cs`, recent migration `20260521174843_LowercaseAllEmails.cs` (4-space EF indent). Frontend — `src/pages/admin/users/admin-users-page.tsx`, `src/components/ui/portal-menu.tsx`, `board-edit-dialog.tsx` (portal modal pattern).

**Conventions (every task):** Backend TABS + file-scoped namespaces + primary-ctor DI (PascalCase params); DTOs `record`; `Guid.CreateVersion7()`; `DateTime.UtcNow`; soft delete; FluentValidation. Frontend no `any`, all text `t()` × 3 locales, TanStack Query, WAAO UI lib. Migrations authored via Bash heredoc (Write/Edit hook-blocked). Commit author `git -c user.email="higor@waao.com.br"`; conventional prefixes; no Claude/AI/Co-Authored-By. Push to `main`.
